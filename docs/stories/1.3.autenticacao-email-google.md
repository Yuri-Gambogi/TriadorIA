# Story 1.3: Autenticação com Email e Google OAuth

## Status

Approved

## Story

**As a** usuário (Yuri e futuros clientes),
**I want** fazer login ou criar conta usando **email/senha** ou **Google OAuth**, com sessão persistindo entre reloads e proteção automática de rotas privadas,
**so that** eu consiga acessar meu espaço pessoal de triagens (que será construído nas próximas stories 1.4 e 1.5) sem que outras pessoas vejam minhas coisas.

## Acceptance Criteria

1. Tela `/login` com formulário de **email + senha** + botão **"Entrar com Google"** + link "Não tem conta? Cadastre-se"
2. Tela `/cadastro` com formulário equivalente (email + senha + nome opcional) + botão Google + link "Já tem conta? Entre"
3. Callback handler em `/auth/callback` (Route Handler) processa o retorno do OAuth e seta cookies de sessão
4. Após login bem-sucedido, usuário é redirecionado pra `/triador` (ou pra `next` se vier no query string `?next=/algumacoisa`)
5. Sessão persiste entre reloads via cookies HTTP-only do Supabase SSR (já validado na Story 1.2)
6. Rotas em `(app)/` (incluindo `/triador`) redirecionam automaticamente pra `/login?next=/triador` se não houver sessão
7. Botão **"Sair"** disponível no `AppHeader`, presente em todas as rotas autenticadas — ao clicar: chama `supabase.auth.signOut()`, limpa cookies, redireciona pra `/`
8. Mensagens de erro claras em pt-BR: email inválido, senha curta (mín 10 chars), credenciais erradas, conta não confirmada, conexão falhou
9. Página `/triador` placeholder (esta story) mostra "Bem-vindo, **{nome ou email}**" + botão Sair. **Substituída pela tela de captura real na Story 1.4.**
10. Componentes Shadcn adicionados: `Input`, `Label`, `Form` (wrapper React Hook Form), `Separator`
11. Google OAuth provider configurado no Supabase Dashboard (Yuri faz)
12. Google Cloud OAuth credentials criadas e redirect URIs configuradas (Yuri faz)
13. Email confirmation **desabilitado** no Supabase pra MVP (Yuri faz — simplifica fluxo de cadastro)
14. `pnpm typecheck`, `pnpm lint`, `pnpm build` zero erros
15. Validado em local: signup + login + Google OAuth + acesso a `/triador` + logout funcionam end-to-end
16. Validado em produção (após push): mesmo fluxo funciona em https://triador-ia.vercel.app

### Middleware atualizado (`middleware.ts`)

```typescript
import { createServerClient } from '@supabase/ssr';
import { NextResponse, type NextRequest } from 'next/server';

const PROTECTED_PREFIXES = ['/triador', '/historico', '/configuracoes'];

export async function middleware(request: NextRequest) {
  let response = NextResponse.next({ request });

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll: () => request.cookies.getAll(),
        setAll: (toSet) => {
          toSet.forEach(({ name, value }) => request.cookies.set(name, value));
          response = NextResponse.next({ request });
          toSet.forEach(({ name, value, options }) => response.cookies.set(name, value, options));
        },
      },
    },
  );

  const { data: { user } } = await supabase.auth.getUser();

  const { pathname } = request.nextUrl;
  const isProtected = PROTECTED_PREFIXES.some((p) => pathname.startsWith(p));

  if (isProtected && !user) {
    const url = request.nextUrl.clone();
    url.pathname = '/login';
    url.searchParams.set('next', pathname);
    return NextResponse.redirect(url);
  }

  return response;
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)'],
};
```

### SignOutButton (`components/layout/SignOutButton.tsx`)

```typescript
'use client';

import { SignOut } from '@phosphor-icons/react';
import { useRouter } from 'next/navigation';
import { getBrowserSupabase } from '@/lib/auth/client';
import { Button } from '@/components/ui/button';

export function SignOutButton() {
  const router = useRouter();

  async function handleSignOut() {
    const supabase = getBrowserSupabase();
    await supabase.auth.signOut();
    router.push('/');
    router.refresh();
  }

  return (
    <Button variant="ghost" size="icon" onClick={handleSignOut} aria-label="Sair">
      <SignOut size={20} weight="bold" />
    </Button>
  );
}
```

### Email confirmation (decisão pro MVP)

[Source: front-end-spec.md#design-principles + decisão pragmática pro MVP]

Por padrão Supabase exige confirmação por email no signup. Pra MVP desabilitamos:

- Supabase Dashboard → Authentication → Providers → **Email** → toggle "Confirm email" **OFF**

Resultado: cadastro com email/senha já loga direto. Sem fluxo de "verifique seu email".

**Tradeoffs documentados:**
- ✅ Fricção zero no signup (perfeito pra MVP single-user)
- ⚠️ Aceita emails inválidos (não verifica que existe)
- ⚠️ Spam protection é menor (mitigar com rate limit do Supabase)

Reativar quando o produto sair do MVP pessoal e tiver usuários externos.
