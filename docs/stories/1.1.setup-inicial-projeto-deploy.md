# Story 1.1: Setup inicial do projeto e deploy

## Status

Done

## Story

**As a** desenvolvedor (Yuri assistido pelo @dev),
**I want** um projeto Next.js inicializado, conectado ao GitHub e com deploy automático no Vercel,
**so that** toda alteração futura seja publicada automaticamente em produção e o time tenha uma base sólida pra construir as próximas stories do Epic 1.

## Acceptance Criteria

1. Repositório Git inicializado em `Triador-AI/` com `.gitignore` apropriado para Next.js (inclui `.next/`, `node_modules/`, `.env.local`, `coverage/`, etc.)
2. Projeto Next.js 15 (App Router) com TypeScript em modo estrito configurado (`strict: true` + `noUncheckedIndexedAccess: true`)
3. Tailwind CSS 4 instalado e configurado, com Shadcn/ui inicializado e pelo menos 1 componente base funcional (`Button`)
4. Phosphor Icons instalado e renderizando ao menos 1 ícone na página inicial (validar regra "Phosphor Icons, não Lucide")
5. Projeto Vercel conectado ao repositório GitHub com deploy automático na branch `main` (push em `main` resulta em deploy produção)
6. Página inicial `/` renderiza "Triador AI — em construção" com tipografia Inter, dark mode toggle funcional e está acessível em URL pública do Vercel
7. README na raiz do projeto com seções: visão geral (1 parágrafo), pré-requisitos (Node 20, pnpm 9), setup local (passo a passo), comandos disponíveis, link pra `docs/prd.md` e `docs/architecture.md`
8. `pnpm dev` inicia o servidor local em `http://localhost:3000` sem warnings/erros
9. `pnpm build` completa com sucesso (sanity check de build de produção)
10. `pnpm lint` e `pnpm typecheck` executam sem erros
